# ZooApplication — консольный зоопарк (Spring Boot + Gradle)
# Тихонов Илья

### 1) Запуск из терминала
```bash
./gradlew bootRun
```

После запуска вы увидите меню:
```
============================
Добро пожаловать в зоопарк!
Выберите действие:
1. Добавить животное
2. Добавить вещь
3. Показать отчёт
0. Выход
============================
Введите номер действия:
```

**Выход** — введите `0` и нажмите Enter.

### 2) Сборка JAR и запуск
Соберите исполняемый Spring Boot JAR:
```bash
./gradlew bootJar
```
Запуск:
```bash
java -jar build/libs/*-SNAPSHOT.jar
```



## Запуск тестов 
Запустить **все тесты**:
```bash
./gradlew test
```
Сформировался отчёт покрытия (JaCoCo), чтобы открыть его в браузере 
перейдите в папку build/reports/jacoco/test/html/index.html

## SOLID

- **S — Single Responsibility:**  
  `ZooService` управляет приёмом животных и инвентарём; `ReportService` только собирает  отчёт `ZooSummaryDto`. Контейнеры (`AnimalList`, `ThingList`) отвечают за хранение коллекций.

- **O — Open/Closed:**  
  Добавление нового вида животного не требует изменений в отчёте: сбор идёт через общий контракт/поведение.

- **L — Liskov Substitution:**  
  `Rabbit`, `Monkey`, `Tiger`, `Wolf` подставимы как животные в сервисах и отчёте — общий интерфейс поведения соблюдён.

- **I — Interface Segregation:**  
  Используются минимально необходимые операции контейнеров (`add`, `findAll`, `clear`, `size`)

- **D — Dependency Inversion:**  
  Сервисы получают зависимости через конструктор (DI), в тестах подменяются моками Mockito.
